# Fix and Improve AI Chat Assistant

## Problem
1. AI chat shows "⚠️ Error processing your request. Please try again."
2. Login page chat (`sendLoginChat`) is not defined
3. AI doesn't work flawlessly with real-time data

## Issues Found

### 1. API Key Missing
- The AI edge function checks for `GEMINI_API_KEY` environment variable (set in Supabase)
- If not set, it returns offline/canned responses (lines 20-32)
- This is likely the primary cause of errors

### 2. Missing Client Functions
- `sendLoginChat()` function is not defined in scripts.js - login page chat won't work
- Floating chat `sendChat()` sends dummy messages only

### 3. Error Handling Issues
- Client doesn't handle API errors gracefully
- No proper error UI feedback for failed requests

### 4. Real-Time Data Gaps
- Context is passed but might be stale
- No live external data (weather, market, news) is integrated

## Plan: Fix & Enhance AI Chat

### PHASE 1: Fix Basic Errors (Quick Wins)

#### 1.1 Add sendLoginChat Function
Add to scripts.js:
```javascript
window.sendLoginChat = async function() {
  const input = $('login-chat-input');
  if (!input?.value.trim()) return;
  const msg = input.value.trim();
  input.value = '';
  
  // Show user message
  const container = $('login-chat-msgs');
  if (container) container.innerHTML += `<div class="chat-msg user">${msg}</div>`;
  
  // Call API
  try {
    const res = await apiCall('/api/ai', {
      method: 'POST',
      body: JSON.stringify({ message: msg, role: 'visitor', context: {} })
    });
    const data = await res.json();
    if (container) container.innerHTML += `<div class="chat-msg bot">${data.message || 'Error'}</div>`;
  } catch (e) {
    if (container) container.innerHTML += `<div class="chat-msg bot">Error: ${e.message}</div>`;
  }
};
```
